# SnowBall X

Frontend-приложение для анализа инвестиционного портфеля (на текущем этапе — облигации):
общий баланс и доходность, календарь дивидендных выплат, учёт вложенных средств и
калькулятор финансовой цели.

Реальный backend БКС пока недоступен, поэтому приложение работает на **моках** через
абстрактный слой API — переключение на реальный источник данных не потребует изменений в UI.

## Возможности

### Экран 1 — Главное
- Общий баланс портфеля, годовой процент пассивного дохода и доход в год.
- Таблица активов с сортировкой: название/тикер, количество, стоимость, дивидендная доходность (TanStack Table).
- Круговая диаграмма долей активов в портфеле (Recharts).
- Кнопки **«Загрузить пример»** (наполнение демо-данными) и **«Обновить»** (рефетч).

### Экран 2 — Календарь дивидендных выплат
- Сводка: всего за год, в среднем в месяц, в день, доходность.
- Столбчатый график выплат по 12 месяцам (цвет столбца — доминирующая категория).
- Календарная сетка (Пн–Вс) с карточками выплат: эмитент, сумма, доходность, статус.
- Навигация по месяцам и сумма выплат за месяц.
- Статусы выплат с легендой: **выплачено / объявлено / прогноз / погашение и амортизация**
  (статус вычисляется от текущей даты).

### Экран 3 — Вложенные средства
- Ручной учёт: форма добавления записи (дата, сумма) с валидацией (React Hook Form + Zod).
- Таблица вложений с удалением строк и общей суммой сверху.
- Данные хранятся локально в `localStorage` и **не синхронизируются** с активами (по ТЗ).

### Экран 4 — Калькулятор цели
- Целевая сумма капитала (ручной ввод).
- Калькулятор с живым пересчётом: ежемесячное пополнение, процент годовых, срок (месяцы/годы),
  чекбокс **«с учётом реинвестирования»** (сложный / простой процент).
- Результат: итоговая сумма, внесено взносами, доход, пассивный доход в месяц/год и срок
  достижения цели.
- График роста капитала с линией цели (Recharts).
- Настройки сохраняются в `localStorage`.

### Общее
- **Неоморфный интерфейс** с переключением светлой/тёмной темы (сохраняется в `localStorage`).
- **Мультиязычность** RU/EN с переключателем (выбор сохраняется).
- Сайдбар-навигация слева.

## Архитектура

Проект следует методологии **Feature-Sliced Design (FSD)**. Зависимости направлены сверху вниз:
`app → pages → widgets → features → entities → shared`.

```
src/
  app/         # провайдеры (Query, Router, Theme, i18n), роутинг, глобальные стили
  pages/       # dashboard, calendar, investments, goal
  widgets/     # portfolio-summary, assets-table, portfolio-pie, dividend-*, goal-calculator, sidebar …
  features/    # theme-toggle, language-switch, load-example-data, refresh-data, add-investment
  entities/    # bond, dividend, investment, goal (модели, расчёты, хуки данных)
  shared/      # api (контракт + mock/bcs + фабрика), ui (неоморфные примитивы), lib, config, i18n
```

### Слой данных (абстракция API)
Приложение зависит только от контракта `PortfolioApi` (`src/shared/api/contract.ts`).
Реализации выбираются фабрикой по переменной окружения `VITE_API_SOURCE`:

- `mock` (по умолчанию) — демо-данные, кнопка «Загрузить пример»;
- `bcs` — заглушка для реального API БКС (`src/shared/api/bcs/bcsApi.ts`).

Когда ключ БКС станет доступен, достаточно дописать `bcsApi` (обмен refresh-токена и маппинг
ответа в тип `Bond`) — UI, фичи и сущности менять не нужно.

### Хранение данных
- `localStorage` — тема, язык, вложенные средства, настройки калькулятора цели, флаг демо-данных.
- Чтение из хранилища проходит через Zod-валидацию (`src/shared/lib/storage.ts`).

## Запуск

Требуется Node.js 18+.

```bash
npm install      # установка зависимостей
npm run dev      # запуск дев-сервера (http://localhost:5173)
npm run build    # проверка типов + production-сборка
npm run preview  # предпросмотр собранного приложения
npm run lint     # линтинг (oxlint)
```

### Переменные окружения (`.env`)
```
VITE_API_SOURCE=mock            # источник данных: mock | bcs (по умолчанию mock)
VITE_BCS_REFRESH_TOKEN=...       # refresh-токен БКС (для будущей интеграции)
```

> `.env` добавлен в `.gitignore` и в репозиторий не попадает.

## Ограничения и допущения
- Приложение рассчитано на одного пользователя, реализация упрощённая (по ТЗ).
- Данные — на моках; реальная интеграция с БКС появится после открытия доступа к API.
- В календаре сознательно отложены режим «Список» и фильтры/поиск — задел под расширение.
